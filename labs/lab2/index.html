    Laboratory Work 2: Основы NumPy (P3124)¶

    1. Goal of the Work¶

    Целью лабораторной работы является освоение библиотеки NumPy для проведения численных вычислений и анализа данных. А также получение практических навыков работы с векторизацией математических операций, матричными вычислениями, статистикой и визуализацией (с использованием matplotlib и seaborn). Дополнительная цель — соблюдение стандартов написания кода (PEP-8, PEP-484, PEP-257) и успешное прохождение модульных тестов pytest.

    2. Theoretical Background¶

    Библиотека NumPy (Numerical Python) — это базовая библиотека для научных вычислений на языке Python. Главным объектом NumPy является многомерный массив ndarray. Векторизация операций в NumPy позволяет применять математические функции ко всем элементам массива без использования явных циклов for (концепция broadcasting), что значительно ускоряет выполнение кода, так как вычисления происходят на низком уровне (C/C++). Ключевые методы, использованные в работе: np.arange(), np.dot(), np.linalg.solve(), np.percentile() и др.

    3. Tasks¶

    1. Создание и обработка массивов: генерация векторов и матриц, изменение формы (reshape) и транспонирование.
    2. Векторные операции: поэлементное сложение, умножение на скаляр, скалярное произведение (dot_product).
    3. Матричные операции: перемножение матриц (через оператор @), вычисление определителя, нахождение обратной матрицы и решение систем линейных уравнений (СЛАУ).
    4. Статистический анализ: загрузка данных из CSV с помощью pandas, расчет среднего, медианы, стандартного отклонения и перцентилей. Min-Max нормализация данных.
    5. Визуализация: построение гистограммы, тепловой карты корреляции и линейного графика с автоматическим сохранением графиков в директорию plots/.

    Реализация матричных вычислений и СЛАУ: 

    import numpy as np

def matrix_multiply(a: np.ndarray, b: np.ndarray) -> np.ndarray:
    """Умножение матриц с использованием оператора @."""
    return a @ b

def solve_linear_system(a: np.ndarray, b: np.ndarray) -> np.ndarray:
    """Решение системы Ax = b через библиотеку линейной алгебры NumPy."""
    return np.linalg.solve(a, b)
    Статистический анализ данных: 
    def statistical_analysis(data: np.ndarray) -> dict:
    """Расчет статистических показателей с явным приведением типов."""
    return {
        "mean": float(np.mean(data)),
        "median": float(np.median(data)),
        "std": float(np.std(data)),
        "min": float(np.min(data)),
        "max": float(np.max(data)),
        "p25": float(np.percentile(data, 25)),
        "p75": float(np.percentile(data, 75))
    }

    5. Conclusion¶

    В результате выполнения лабораторной работы были закреплены практические навыки работы с библиотекой NumPy, а также базовой интеграции с Pandas, Matplotlib и Seaborn. Главным итогом стало глубокое понимание принципов векторизации вычислений, что позволяет писать эффективный и быстрый код, избегая ресурсоемких циклов for.

    С какими проблемами столкнулись и как их решили: 1. Специфичная типизация данных NumPy: При добавлении аннотаций типов (PEP-484) возникла проблема с тем, что встроенные агрегирующие функции (например, np.mean или np.percentile) возвращают тип numpy.float64, а не стандартный Python float. Решением стало явное приведение типов с помощью обертки float(), что обеспечило корректную работу и соответствие заявленным аннотациям. 2. Ошибки файловой системы при тестировании: Во время работы функций визуализации тесты могли завершаться с ошибкой FileNotFoundError, так как скрипт пытался сохранить графики (plt.savefig) в несуществующую папку plots/. Эта проблема была устранена путем добавления проверки и автоматического создания директории с помощью os.makedirs("plots", exist_ok=True). 3. Путаница между операторами умножения: При работе с матрицами было критически важно различать поэлементное умножение и классическое матричное. Проблема решалась строгим использованием оператора @ (или np.matmul) для матриц и * для векторов.

    В конечном итоге, все функции были реализованы корректно. Итоговый код соблюдает стандарты DAST/DATS (PEP-8, PEP-257) и успешно проходит все 17 подготовленных модульных тестов pytest.
