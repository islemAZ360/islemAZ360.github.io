# Лабораторная работа №3: Автоматизация развертывания статического сайта

## 1. Цель работы
Реализовать сценарий автоматического развертывания статического сайта, построенного на движке **MkDocs**, одновременно на две платформы: **GitHub Pages** и **Sourcecraft**.

## 2. Выполненные действия
1.  **Инициализация проекта**: Создан локальный репозиторий с проектом MkDocs.
2.  **Настройка удаленных репозиториев**:
    *   Добавлен репозиторий GitHub как `origin`.
    *   Добавлен репозиторий Sourcecraft с использованием персонального токена доступа (PAT).
3.  **Автоматизация (CI/CD)**:
    *   Создан Workflow для **GitHub Actions** для автоматической сборки и деплоя при пуше в ветку `main`.
    *   Настроена конфигурация **Sourcecraft Sites** для отображения сайта из ветки `main`.
4.  **Синхронизация**: Код был успешно отправлен в оба репозитория с помощью команд `git push origin main` и `git push sourcecraft main`.

## 3. Организация репозитория
Репозиторий имеет следующую структуру:
*   `.github/workflows/deploy.yml`: Сценарий сборки и деплоя для GitHub.
*   `.sourcecraft/sites.yaml`: Конфигурационный файл для хостинга на Sourcecraft.
*   `source/docs/`: Исходные файлы документации в формате Markdown.
*   `source/mkdocs.yml`: Основной файл настроек MkDocs.
*   `source/site/`: Скомпилированные HTML-файлы (используются для Sourcecraft).

## 4. Процесс деплоя
Для выполнения деплоя необходимо выполнить следующие шаги:
1.  Внести изменения в файлы документации.
2.  Выполнить сборку сайта (локально или через экшены).
3.  Отправить изменения в GitHub: `git push origin main`.
4.  Отправить изменения в Sourcecraft: `git push sourcecraft main`.

## 5. Настройки репозиториев
### GitHub:
*   В разделе **Settings > Pages** выбрано использование **GitHub Actions** в качестве источника сборки.
*   Настроен секретный токен для доступа экшена к репозиторию (при необходимости).

### Sourcecraft:
*   **Персональный токен (PAT)**: Создан токен с правами `Maintainer` для HTTPS доступа.
*   **Configurations > Sites**: Создана конфигурация, указывающая на `root: "source/site"` и ветку `ref: "main"`, что позволяет платформе отображать статические файлы напрямую.
