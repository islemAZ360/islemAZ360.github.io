# Лабораторная работа №3: Автоматизация развертывания статического сайта

## 1. Цель работы
Реализовать сценарий автоматического развертывания статического сайта, построенного на движке **MkDocs**, одновременно на две платформы: **GitHub Pages** и **Sourcecraft**.

## 2. Выполненные действия
1.  **Инициализация проекта**: Создан локальный репозиторий с проектом MkDocs.
2.  **Настройка удаленных репозиториев**:
    *   Добавлен репозиторий GitHub как `origin`.
    *   Добавлен репозиторий Sourcecraft с использованием персонального токена доступа (PAT).
3.  **Автоматизация (CI/CD)**:
    *   Создан Workflow для **GitHub Actions** для автоматической сборки и деплоя при пуше в ветку `main`.
    *   Создан пайплайн **Sourcecraft CI** (`ci.yaml`) для независимой сборки сайта на стороне платформы Sourcecraft и пуша в ветку `release`.
    *   Настроена конфигурация **Sourcecraft Sites** для отображения сайта из ветки `release`.

## 3. Организация репозитория
Репозиторий имеет следующую структуру:
*   `.github/workflows/deploy.yml`: Сценарий сборки и деплоя для GitHub.
*   `.sourcecraft/ci.yaml`: Сценарий CI/CD для автоматической сборки сайта на серверах Sourcecraft.
*   `.sourcecraft/sites.yaml`: Конфигурационный файл для хостинга на Sourcecraft.
*   `requirements.txt`: Файл с зависимостями Python (mkdocs, mkdocs-material) для сборки.
*   `source/`: Папка с исходными файлами документации и файлом настроек `mkdocs.yml`.

## 4. Основной процесс деплоя
Деплой происходит полностью автоматически:
1.  При выполнении `git push origin main`, GitHub Actions собирает сайт и публикует его в ветку `gh-pages`.
2.  При выполнении `git push sourcecraft main`, Sourcecraft CI собирает сайт по инструкциям из `ci.yaml`, коммитит готовую статику в ветку `release`, после чего Sourcecraft Sites автоматически отображает сайт из папки `site` этой ветки.

## 5. Альтернативный метод деплоя (Синхронизация веток)
В качестве одного из способов выполнения лабораторной работы (вместо независимой сборки на Sourcecraft) можно использовать сборку на стороне GitHub с последующим переносом готовой статики. Для этого используется следующая команда:

`git push sourcecraft origin/gh-pages:refs/heads/gh-pages --force`

**Описание:** Эта команда копирует готовую ветку с сайтом (`gh-pages`) с GitHub напрямую на Sourcecraft. После этого достаточно настроить `sites.yaml` на чтение из ветки `gh-pages`. Это эффективный метод, если мы хотим использовать только один CI/CD инструмент (GitHub Actions).
